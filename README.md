# SokuSai

推しの休止期間中や活動終了後などに元気で生きてほしいという気持ちを行動に移す。表舞台に出ていないときでも誰かが自分のことを想っていてくれているということを実感できるようにしたい。

推しの名前を書いた紙を紙飛行機として折り、青空に向けて飛ばす体験がサービスの根幹価値であり、推される側に対しては表舞台に立っていなくとも自分のことを想ってくれているひとがいるのだと活動の意味を実感できるような体験を提供する。

ブラウザベースのWebサービスとして企画を進めたい。

ファンの利用シーンについては、起床したときと仕事終わりにSokuSaiにアクセスするイメージで進めてください。スタンプカードのデザインをファンの間で共有して毎日スタンプラリーができる体験や記念日を登録して記念日には淡い水色の特別な折り紙を普段の折り紙の代わりとして紙飛行機にできる体験も追記してください。青空の空間デザインについては、他のユーザーが紙飛行機を飛ばしたことがトリガーになってスマホの画面の上半分あたりの右側から左側にかけて紙飛行機がリアルタイムで飛んでいくイメージで進めてください。

大前提として、アカウントはツイッターやグーグルのアカウントへのログインを基本としてサービスを構築する方向で進めてください。推し側の想いの受け取り方については、紙飛行機をクリックすると開くことができ、そこには「今日も（ファンの名前）さんが紙飛行機を飛ばしてくれました」と書かれている、というユーザー体験をイメージしています。メッセージ機能については、推しの名前以外のメッセージは考えていません。推しのツイッターアカウントをニックネーム登録しておいてそのニックネームをアクセス時に宛名として入力するサービス設計にしたいです。スタンプカードのデザイン共有のイメージについては、Figmaのコミュニティ投稿のようにイラスト投稿主がスタンプカードの利用を許諾する形で共有するイメージです。自分がイラストの著作権所有者であると誓約してもらうことも必要ですね。

推し本人の認証（本人確認）については、推し本人がそのTwitterアカウントでSokuSaiにログインすれば、自動的に自分宛ての紙飛行機が集まるダッシュボードに入れるという設計にしてください。サービスの持続可能性（マネタイズ）については、基本無料で、記念日以外の「特別な柄の折り紙」を有料（買い切り）で販売することに加えて、サービスへの建白書を有料で送れることもマネタイズの一貫として機能実装したいです。 紙飛行機の蓄積については、「今日は〇〇機の紙飛行機が届いています」というカウンターが推し側の画面にあるイメージで進めてください。

技術スタックについてはバックエンドにGo言語を採用し、フロントエンドにTypeScriptを採用する方針で進めてください。
